Workflow séquentiel
STEP 01
Idée → Spec
/jo "j'ai une idée : …"
JO lit le contexte, challenge avec 3 questions, itère avec le Talent jusqu'à validation.
→ specs/feature_[ID].md
statut: VALIDÉE TALENT
STEP 02 opt.
Spec → Figma
/design-workflow "spec feature_[ID]"
Bridge DS génère le frame Figma depuis la spec. Recommandé pour features visuellement complexes.
→ Frame Figma annoté
STEP 03
Spec → Code
/bob "implémente feature_[ID]"
BOB lit la spec + design_guide + frame Figma. Code en Ralph Loop itératif (6 étapes).
→ /app · /components
Rapport conformance
STEP 04
Code → Verdict
/do "évalue feature_[ID]"
DO évalue sur 4 dimensions, rend un score /20 et des feedbacks actionnables vers BOB ou JO.
→ Score /20 + Verdict
