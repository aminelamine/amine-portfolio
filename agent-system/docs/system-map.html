Workflow séquentiel
1
JO · PLAN
→ challenge + risques
→ specs/feature_[ID].md
→ statut: VALIDÉE
2
Bridge · DESIGN
→ lit la spec JO
→ frame Figma + tokens
→ si feature complexe
3
BOB · SHIP
→ lit spec + Figma
→ code Next.js / TS
→ PR ready
4
DO · ANALYZE
→ score /20
→ verdict + feedbacks
→ → retour JO si <14
Feedback loop : si le score DO est inférieur à 14/20, la feature repasse en phase PLAN — JO re-challenge la spec avant tout nouveau cycle BOB.
Règles invariantes
🚫 Aucun code sans spec au statut VALIDÉE TALENT
🔒 Jamais modifier /components/ui/ — ownership Shadcn total
📐 Composants <150 lignes — découper si dépassement
⚡ TypeScript strict : zéro any, zéro @ts-ignore
📚 Aucune lib UI hors design_guide.md sans validation explicite
🎨 Esthétique distincte : fonts uniques, palette cohérente, motion intentionnel
